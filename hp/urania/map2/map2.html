INSTRUCTIONS:
1. Download the package and install the files on your HP48GX.
   MAP2RES.lib can be stored in every port, whereas MAP2.lib must be stored in
   port 0 or 1.
2. You must go in the directory where all the settings for Urania48
   are stored (Zone, Alt, Long, Lat).
3. Press [MAP2] for entering the application,


FIRST CHOOSE BOX:

  New conditions: for drawing a new map,

  Load map: for loading a saved map from memory.

LOAD MAP:

  choose the correct variable using the browser, .. is for going up.
  When ready press OK or ENTER.

NEW CONDITIONS:

  just set the correct time and date,
  you can change the position settings by pressing POS,

  SIZE control the radius of the map in pixels,
  STAR # is for the number of star to plot,
  PLANETS enables the drawing of solar system objects,
  MESSIER enables the drawing of Messier Objects,
  Press OK when ready.

  it takes approximatively 8min for a plotting all the objects (1504
  stars, Messier and Planets),

KEYMAP

  the cursor moving is controlled by the four arrows, and by the
  numeric keys (except 5), the vector from 5 to the key give you the
  direction of the moving (for instance 8 is up).
  LS repeat the moving 25 times in the choosen direction.

  [5]   toggles name printing,

  [+/-] toggles coordinates printing in Alt/Az form, setting this option
      slow down considerably the moving of the cursor,
 
  [X]   toggles zoom mode (not x but times)

      Keymap in zoom mode
  
      + enlarge the zoom circle,
      - reduce the zoom circle,
      CANCEL & X : leave zoom mode,
      ENTER: select zoom

  [A]   call the Urania info sheet on the object pointed by the cursor
      (Planets, Moon, Sun or Messier). Press CANCEL for returning to
      the map.

  [.]   flash-print the coordinates of the cursor,

  [CANCEL] unzoom, or leave the map,

  [STO] stores the map in memory, enter a valid variable name,

  [+]   zoom in (only works for zoomed map),

  [-]   zoom out (only works for zoomed map),

  [RS]-<dir> move the zoom window in the choosen direction, the center
      of current window is placed at the edge of the new one.
      It doesn't work for diagonal move.
